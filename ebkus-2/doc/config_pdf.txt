Installationshinweise zur Verwendung der PDF-Formate in EBKuS
=============================================================

1. Server
=========

Das PDF-Package reportlab.tgz bzw. reportlab.zip muss im Verzeichnis
/ebkus/reportlab vorhanden sein, sollen die Aktendokumente im PDF-Format
zum Lesen ausgegeben werden.

Es ist im Internet kostenlos zu beziehen unter:
http://www.reportlab.com/download.html

Das Archiv wird in das Verzeichnis /ebkus/reportlab ausgepackt.


2. Klient-PC
============

Hier muss der "Acrobat Reader" von Adobe installiert und beim
verwendeten Browser als "Plugin" oder "Helper Application"
konfiguriert sein.

Der "Acrobat Reader" ist im Internet zu beziehen unter:
http://www.adobe.com/products/acrobat/readermain.html

Bei den bekannten Linux-Distributionen ist der Acobat Reader meistens
dabei. Er kann immer als "Helper Application" beim Netscape
eingetragen werden (Automatischer Start des Readers ausserhalb des
Browsers, wenn eine PDF-Datei übertragen werden soll).

Die Verwendung des Readers als Plugin bei Netscape höher 4.0 sollte
mit dieser (glibc6-gelinkten) Adobe-Version möglich sein:
http://rpmfind.net/linux/RPM/contrib/libc6/i386/acroread-4.05-3.i386.html


3. Konvertieren ins PDF-Format mit StarOffice unter Unix
========================================================

Man richte in StarOffice einen neuen Drucker ein, z.B. "pdf_queue",
der eine PDF_Datei erzeugt. Die Default-Postscript-Ausgabe von
StarOffice muss dazu an Ghostscript ( mit den Optionen:
-sDEVICE=pdfwrite -sOutputFile=file.pdf -q -DNOPAUSE ) weitergeleitet
werden. Die Drucker bei StarOffice sind in der Datei: "Xpdefaults"
definiert, mit Beispielen.


4. Konvertierung ins PDF-Format unter Windows
=============================================

Hinweise zum Konvertieren ins PDF-Format unter Windows finden Sie
unter: http://www.wernerroth.de/staroffice/dokus/postscri.html
